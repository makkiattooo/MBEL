# MBEL：ヒントとベストプラクティス

MBEL を最大限に活用するために、これらのパターンに従ってください。

## 1. 命名規則
`label1` のようなフラットな名前ではなく、階層的なキーを使用してください。
*   **パターン**: `[feature].[screen].[component].[element]`

## 2. ファイルの構成
ローカライズを論理的な単位に分割してください。フォルダ構造は `--ns` オプションによって自動的に名前空間を定義します。

## 3. AI コンテキストの活用
`@AI_Context` を惜しまないでください。これは自動翻訳の品質保証です。

## 4. ロジックブロック vs コード内での補間
Go コード内で文章を組み立てるのを避けてください。
*   **悪い例 (Go)**: `fmt.Sprintf(mbel.T(ctx, "hello") + " " + name)`
*   **良い例 (MBEL)**:
    ```mbel
    welcome(name) {
        [other] => "ようこそ、{name} さん！"
    }
    ```
