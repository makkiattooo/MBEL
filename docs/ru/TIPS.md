# MBEL: Советы и лучшие практики

Следуйте этим паттернам, чтобы получить максимум от MBEL.

## 1. Соглашения об именовании
Используйте иерархические ключи вместо плоских имен вроде `label1`.
*   **Паттерн**: `[feature].[screen].[component].[element]`

## 2. Организация файлов
Разделяйте локализации на логические единицы. Структура папок автоматически определяет пространства имен с помощью опции `--ns`.

## 3. Использование ИИ-контекста
Не экономьте на `@AI_Context`. Это страховка качества ваших автоматизированных переводов.

## 4. Логические блоки vs Интерполяция в коде
Избегайте сборки предложений в коде Go.
*   **Плохо (Go)**: `fmt.Sprintf(mbel.T(ctx, "hello") + " " + name)`
*   **Хорошо (MBEL)**:
    ```mbel
    welcome(name) {
        [other] => "Добро пожаловать, {name}!"
    }
    ```
